# Codex CI/CD Governance Copilot

Codex-first, human-governed CI/CD review system for enterprise teams.

The main artifact is a **deployment review** generated by Codex SDK and enforced by deterministic policy gates.

## Core idea

- Codex SDK (`@openai/codex-sdk`) is the primary reasoning engine.
- Deterministic governance policy remains the final authority for block/allow decisions.
- Human approval is mandatory before execution.
- Adaptive approval quorum enforces two-person approval for high/critical risk deployments.
- Separation-of-duties blocks requester self-approval.
- MCP tools, App Server endpoints, and CI PR comments all use the same review pipeline.

## Architecture

1. Input deployment request (`repo`, `branch`, `env`, `changedFiles`, commands, test status).
2. Deterministic policy evaluation computes violations and highest severity.
3. Codex SDK produces narrative + checklist + recommendation.
4. Severity threshold gate (`--fail-on`) decides `finalDecision` (`block` or `approve_with_human_gate`).
5. Approval packet is created with risk level and required approvers.
6. Human approvers explicitly approve packet (`token=approve`) with no self-approval allowed.
7. Deploy executes only when approval quorum is met.
6. Full audit trail records policy, Codex review, approval, and execution events.

## Environment

Set these in your shell or `.env`:

```bash
set CODEX_API_KEY=your_key_here
set CODEX_MODEL=gpt-5.3-codex
set BREAK_GLASS_APPROVERS=sre-lead,platform-lead,incident-manager
```

Fallback behavior: if API key is missing/unavailable, deterministic explanation is produced and safety gates still run.

## Commands

Install:

```bash
npm install
```

Codex-first review (main command):

```bash
npm run policy:review -- --repo payments --branch main --env staging --deploy "npm run deploy:staging" --rollback "npm run rollback:staging" --tests true --change-window true --changed-files "src/a.ts,infra/main.tf" --fail-on high --by local-ci
```

Emergency break-glass review (requires incident ticket + senior approvers):

```bash
npm run policy:review -- --repo payments --branch feature/infra-risk --env prod --deploy "npm run deploy:prod" --rollback "npm run rollback:prod" --tests true --change-window false --changed-files "infra/main.tf" --break-glass true --incident-ticket "INC-2026-001" --fail-on high --by incident-commander
```

Legacy policy-only output (backward-compatible):

```bash
npm run policy:check -- --repo payments --branch main --env staging --deploy "npm run deploy:staging" --rollback "npm run rollback:staging" --tests true --change-window true --changed-files "src/a.ts" --fail-on high --by local-ci
```

Codex explanation only:

```bash
npm run policy:explain -- --repo payments --branch main --env staging --deploy "npm run deploy:staging" --rollback "npm run rollback:staging" --tests true --change-window true --changed-files "src/a.ts" --fail-on high --by local-ci
```

Approval and deploy:

```bash
npm run dev -- plan --repo payments --branch main --env prod --deploy "npm run deploy:prod" --rollback "npm run rollback:prod" --tests true --change-window true --changed-files "deploy/prod.yml" --by alice
npm run dev -- approve --packet <packetId> --approver techlead --reason "validated controls" --token approve
npm run dev -- approve --packet <packetId> --approver sre-lead --reason "second reviewer" --token approve
npm run dev -- packet --packet <packetId>
npm run dev -- deploy --packet <packetId>
```

Visible governance dashboard:

```bash
npm run governance:summary
```

App Server + MCP:

```bash
npm run start:app
npm run start:mcp
```

## GitHub workflow

Workflow: `.github/workflows/policy-gate.yml`

- Collects changed PR files
- Runs `policy:review` (Codex + policy gate)
- Posts/updates a single PR governance comment with:
  - status, final decision, threshold
  - policy violations
  - Codex narrative and checklist
- Fails workflow only after comment is posted
